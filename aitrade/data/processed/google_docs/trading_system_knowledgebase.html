Trading System Principles

trading #principles

The goal is not to make the machine clever. The goal is to make the machine hard to fool.

First Principles

Markets are noisy. Most signals are weak. Execution mistakes are expensive. So the system must be boring.

Good automation has three properties:

  1. It can explain what it saw.
  2. It can explain what it decided.
  3. It can refuse to act.

Invariants

  • Broker execution is not the source of truth for strategy data.
  • Layer 1 never trades.
  • Every run writes local artifacts.
  • Every external provider can fail without killing the whole system.
  • Real-money orders require explicit confirmation unless a future written policy says otherwise.
  • The agent account is the only account in scope.

Bias

Prefer:

  • tables over charts
  • JSON over screenshots
  • one command over many clicks
  • small modules over one giant script
  • local artifacts before cloud dashboards
  • simple signals before machine learning

Avoid:

  • fancy visuals that do not feed a decision
  • hidden state
  • broker-dependent strategy logic
  • inverse or leveraged ETFs as a first version
  • strategies that cannot be explained in one paragraph

Mental Model

sentiment = why
trend = when
capital flow = where pressure is already visible
risk = how much
execution = hands
logs = memory

The strategy is allowed to be simple. The safety layer is not.

Layer Boundaries

Layer 1 observes:

  • news/event headlines
  • market bars
  • sector sentiment
  • capital-flow pressure

Layer 2 decides:

  • buy candidate
  • exit candidate
  • avoid candidate
  • cash-only state

Layer 3 sizes:

  • account equity
  • buying power
  • risk per trade
  • total portfolio risk

Layer 4 executes:

  • broker quote refresh
  • order review
  • user confirmation
  • order placement

Layer 5 remembers:

  • artifacts
  • logs
  • fills
  • decisions
  • errors

Why This Shape

If a signal is wrong, we inspect Layer 1 and Layer 2.

If a size is wrong, we inspect Layer 3.

If a broker action is wrong, we inspect Layer 4.

No mystery soup.

Provider Rule

Providers are unreliable.

Therefore provider errors become warnings, not fatal crashes. A partial snapshot is better than no snapshot, as long as the warning is explicit.

Example:

  "warnings": [
    "GDELT fetch failed: rate limit"

}

Capital Flow Proxy

trading #capital-flow

We do not have true institutional fund-flow data. So we build a humble proxy and label it honestly.

Old Version

The old project made Sankey charts. Useful for exploration. Too visual for automation.

The automation needs a table.

Current Proxy

relative_flow_score = price_change_pct * average_dollar_volume

Interpretation:

  • positive score: liquid ETF moved up over the window
  • negative score: liquid ETF moved down over the window
  • larger absolute score: stronger visible pressure

This is not true net flow. It is pressure.

Artifact

/Users/r2k8/Library/CloudStorage/OneDrive-Personal/CODE/trading-system/data/processed/capital_flows.csv

Columns:

rank
symbol
name
as_of
direction
price_change_pct
dollar_volume
relative_flow_score

How Strategy Should Use It

Capital flow is a filter, not a standalone trade.

Good:

positive sentiment + positive flow + EMA20 > EMA50 = stronger long candidate

Also good:

negative sentiment + negative flow = avoid or exit candidate

Bad:

positive flow alone = buy

Why Keep It

Sentiment can be early but noisy.

Trend can be late but stable.

Capital-flow pressure tells us whether the market is already voting with money-like behavior.

The signal gets interesting when all three agree.

Glossary

trading #glossary

Artifact

A file written by a layer. Usually JSON or CSV. It is inspectable and can be consumed by the next layer.

Capital-Flow Proxy

A simple pressure score:

price_change_pct * average_dollar_volume

Not true fund flow. Useful as a ranking feature.

Confidence

How much the system trusts the score. Confidence should combine model confidence and sample size.

Layer 1

Observation layer. Fetches data, scores it, writes artifacts. Does not trade.

Regime

The market environment:

  • risk-on
  • neutral
  • defensive
  • crash-watch

Signal

A proposed action. Signals are not orders.

Order

A broker instruction. Orders are real money and require stricter rules.
